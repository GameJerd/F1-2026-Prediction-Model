# F1-2026-Prediction-Model

Focused FastF1 + scikit-learn pipeline for predicting:
- 2026 Chinese GP qualifying top 10
- 2026 Chinese GP race order

Main script: `china_2026_fastf1_pipeline.py`

## What This Pipeline Does
The script runs in two phases:
1. Prepare dataset from FastF1 sessions (2023-2026), engineer features, and save to CSV.
2. Train qualifying and race models, then generate 2026 China predictions.

Outputs generated by the script:
- `china_training_dataset_2023_2025.csv`
- `china_2026_predicted_qualifying_top10.csv`
- `china_2026_predicted_race_order.csv`

## Features Included
- Qualifying and race lap/sector pace metrics
- Driver and team rolling form (last 5 races)
- Shanghai historical aggregates per driver
- Session weather averages (air temp, track temp, humidity, wind, rainfall)
- Tire degradation proxy from stint lap-time slope

## Requirements
- Python 3.10+
- `fastf1`
- `numpy`
- `pandas`
- `scikit-learn`

Install dependencies:

```bash
pip install fastf1 numpy pandas scikit-learn
```

## Usage
From the repository root:

Run full pipeline (prepare + train + predict):

```bash
python china_2026_fastf1_pipeline.py --phase all
```

Prepare dataset only:

```bash
python china_2026_fastf1_pipeline.py --phase prepare
```

Train/predict using existing saved dataset:

```bash
python china_2026_fastf1_pipeline.py --phase train
```

Optional flags:
- `--max-events-per-year <int>` limits fetched events per season (default: 6)
- `--use-telemetry` enables FastF1 telemetry downloads (slower, heavier)

## Notes
- FastF1 cache is written to `f1_cache/`.
- The pipeline is tuned for China-specific prediction context.
- Runtime depends on API fetch volume and local cache state.
